---
title: "config"
description: "MineClawd-Einstellungen oeffnen, lesen oder aktualisieren."
---

## Zweck

Nutze diesen Befehl, um die GUI-Konfiguration zu oeffnen oder Einstellungen im Chat zu verwalten.

## Syntax

```bash
/mineclawd config
/mineclawd config <key>
/mineclawd config <key> <value>
```

## Haeufige keys

- `provider`: `openai` oder `vertex-ai`
- `endpoint`, `api-key`, `model`, `summarize-model`
- `vertex-endpoint`, `vertex-api-key`, `vertex-model`, `vertex-summarize-model`
- `debug-mode`: `true` oder `false`
- `limit-tool-calls`: `true` oder `false`
- `tool-call-limit`: `1` bis `20`
- `system-prompt`
- `tavily-api-key`: Tavily API-Key fuer Websuche
- `broadcast-requests-to`: `self`, `all` oder `ops`
- `dynamic-registry-mode`: `auto`, `enabled` oder `disabled`

## Beispiele

```bash
/mineclawd config provider openai
/mineclawd config api-key sk-...
/mineclawd config tool-call-limit 12
/mineclawd config broadcast-requests-to ops
/mineclawd config dynamic-registry-mode enabled
```

## Verhalten und Grenzen

- `/mineclawd config` oeffnet die GUI nur, wenn der Spieler-Client MineClawd installiert hat.
- `broadcast-requests-to` gilt pro Spieler und muss im Spiel gesetzt werden.
- Geheime Werte werden beim Auslesen maskiert.
- Das Aendern von `dynamic-registry-mode` erfordert einen Neustart von Spiel oder Server.

## Haeufige Fehler

- `unknown config key`: Rechtschreibung pruefen oder Tab-Completion nutzen.
- `invalid boolean`: nutze `true` oder `false`.
- `tool-call-limit must be between 1 and 20`: im gueltigen Bereich bleiben.
