## Zweck

Diese Seite zeigt dir das vollständige MineClawd-Konfigurationsmodell, einschließlich Dateieinstellungen, Befehlsschlüsseln und Neustartverhalten.

## Wo Einstellungen gespeichert sind

- Globale Konfigurationsdatei: `config/mineclawd.json5`
- Chat-Sichtbarkeit pro Spieler: `mineclawd/player-settings.json`
- Soul-Dateien und aktiver Soul-Status: `mineclawd/souls/` und `mineclawd/souls/.active/`

## Vollständiges `mineclawd.json5`-Template

```json5
{
  provider: "OPENAI",
  endpoint: "https://api.openai.com/v1",
  apiKey: "",
  model: "gpt-4o-mini",
  summarizeModel: "gpt-4o-mini",

  vertexEndpoint: "https://aiplatform.googleapis.com/v1",
  vertexApiKey: "",
  vertexModel: "publishers/google/models/gemini-3-pro-preview",
  vertexSummarizeModel: "publishers/google/models/gemini-3-pro-preview",

  debugMode: false,
  limitToolCalls: false,
  toolCallLimit: 16,
  systemPrompt: "",
  dynamicRegistryMode: "AUTO"
}
```

<Info>
In Dateiform sind Enum-Werte großgeschrieben (`OPENAI`, `VERTEX_AI`, `AUTO`, `ENABLED`, `DISABLED`).

In Befehlsform sind Werte kleingeschrieben (`openai`, `vertex-ai`, `auto`, `enabled`, `disabled`).
</Info>

## Schlüsselreferenz

| Dateischlüssel | Befehlsschlüssel | Typ | Standard | Hinweise |
| --- | --- | --- | --- | --- |
| `provider` | `provider` | enum | `OPENAI` | Provider-Auswahl: OpenAI-kompatibel oder Vertex AI. |
| `endpoint` | `endpoint` | string | `https://api.openai.com/v1` | OpenAI-kompatible Basis-URL, kein abschließender Slash erforderlich. |
| `apiKey` | `api-key` | string | leer | OpenAI-API-Key. |
| `model` | `model` | string | `gpt-4o-mini` | OpenAI-Chat-Modell. |
| `summarizeModel` | `summarize-model` | string | `gpt-4o-mini` | OpenAI-Modell für die Titelgenerierung der Session im ersten Turn. |
| `vertexEndpoint` | `vertex-endpoint` | string | `https://aiplatform.googleapis.com/v1` | Vertex-API-Basis-URL. |
| `vertexApiKey` | `vertex-api-key` | string | leer | Vertex-API-Key. |
| `vertexModel` | `vertex-model` | string | `publishers/google/models/gemini-3-pro-preview` | Vertex-Modellpfad oder Modellname. |
| `vertexSummarizeModel` | `vertex-summarize-model` | string | `publishers/google/models/gemini-3-pro-preview` | Vertex-Summarize-Modell für Session-Titel. |
| `debugMode` | `debug-mode` | boolean | `false` | Protokolliert detaillierte Anfrage- und Tool-Aktivität. |
| `limitToolCalls` | `limit-tool-calls` | boolean | `false` | Aktiviert ein Tool-Loop-Limit pro Anfrage. |
| `toolCallLimit` | `tool-call-limit` | integer | `16` | Erlaubter Bereich ist `1` bis `20`. |
| `systemPrompt` | `system-prompt` | string | leer | Leer bedeutet eingebauter System-Prompt. Nutze im Befehlsmodus den Wert `default`, um zurückzusetzen. |
| `dynamicRegistryMode` | `dynamic-registry-mode` | enum | `AUTO` | Steuert den Runtime-Modus für dynamische Platzhalter. Nach Änderung ist ein Neustart erforderlich. |

## Einstellung pro Spieler

`broadcast-requests-to` ist nur per Befehl verfügbar und gilt pro Spieler.

- Befehlsschlüssel: `broadcast-requests-to`
- Erlaubte Werte: `self`, `all`, `ops`
- Gespeichert in: `mineclawd/player-settings.json`
- Zweck: steuert, wer Prompt-Echo und Start-/Endzeilen von Aufgaben sieht.

## Verhalten des Dynamic-Registry-Modus

- `AUTO`: in der Client-Runtime (Singleplayer) aktiviert, auf dedizierten Servern deaktiviert.
- `ENABLED`: in allen Runtimes erzwungen aktiviert.
- `DISABLED`: vollständig deaktiviert.

<Warning>
Wenn du auf einem dedizierten Server `ENABLED` erzwingst, müssen beitretende Clients MineClawd installiert haben.
</Warning>

## Empfohlene Baselines

### Baseline für das erste OpenAI-Setup

```bash
/mineclawd config provider openai
/mineclawd config endpoint https://api.openai.com/v1
/mineclawd config api-key <your-key>
/mineclawd config model gpt-4o-mini
/mineclawd config summarize-model gpt-4o-mini
```

### Baseline für das erste Vertex-Setup

```bash
/mineclawd config provider vertex-ai
/mineclawd config vertex-endpoint https://aiplatform.googleapis.com/v1
/mineclawd config vertex-api-key <your-key>
/mineclawd config vertex-model publishers/google/models/gemini-3-pro-preview
/mineclawd config vertex-summarize-model publishers/google/models/gemini-3-pro-preview
```

## GUI-Weg

- Im Spiel: **Mod Menu** -> **MineClawd** -> **Configure**
- Befehl als Fallback: `/mineclawd config ...`

Wenn das Öffnen der Client-GUI fehlschlägt, konfiguriere über die Befehlsschlüssel.
