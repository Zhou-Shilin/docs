---
title: "sessions resume"
description: "Die aktive Session per ID oder Token wechseln."
---

## Zweck

Mache eine ältere Session wieder aktiv, damit neue Anfragen in diesem Kontext fortgesetzt werden.

## Syntax

```bash
/mineclawd sessions resume <session>
```

`<session>` kann sein:

- Session-ID, wie `1a2b`
- Session-Token, wie `1a2b-my-project-title`

## Beispiele

```bash
/mineclawd sessions resume 1a2b
/mineclawd sessions resume 1a2b-my-project-title
```

## Verhalten und Grenzen

- Das Fortsetzen bearbeitet den Verlauf nicht. Es wechselt nur den aktiven Session-Zeiger.
- Nutze zuerst `/mineclawd sessions list`, wenn du nicht sicher bist, welche Referenz du verwenden sollst.

## Häufige Fehler

- `session not found`: exakte ID oder exaktes Token aus der Listenausgabe verwenden.
- `cannot switch sessions while a request is running`: auf Abschluss warten.
