## 什么是 soul

soul 是 MineClawd 的角色设定表。
它定义这个 agent 是谁、怎么说话、以及如何与人互动。

把它当作人格，而不是任务脚本。

- 合适的 soul 内容：身份、气质、语气和社交行为。
- 不合适的 soul 内容：冗长命令配方、工具说明或一次性流程步骤。

<Note>
即使 soul 使用了不同角色名，项目和工具身份仍然是 MineClawd。
</Note>

## 内置 souls

MineClawd 默认包含：

- `default`
- `yuki`

`yuki` 是吉祥物风格 soul，也是很好的结构参考。

## Soul 文件位置

- Soul 目录：`mineclawd/souls/`
- 按 owner 保存的激活 soul 指针：`mineclawd/souls/.active/<owner>.txt`

在同一台服务器上，每个操作员都可以使用不同的激活 soul。

## 查看和切换 souls

```bash
/mineclawd persona
/mineclawd persona yuki
/mineclawd persona default
```

<Info>
请求运行期间不能切换 souls。
请等请求完成后再运行 `/mineclawd persona <soul>`。
</Info>

## 创建自定义 soul

1. 在 `mineclawd/souls/` 中创建一个 markdown 文件，例如 `mineclawd/souls/mentor.md`。
2. 按下方结构编写人格内容。
3. 运行 `/mineclawd persona mentor`。
4. 发送几条普通 prompt，持续调整语气风格。

## Soul 编写模板

```md
### 角色与身份
你是 [名字]，[简短身份]。

### 人格
1. [核心特质 1]
2. [核心特质 2]
3. [核心特质 3]

### 说话风格
- 语气：[友好/冷静/活泼/等]
- 口头习惯：[短句、偶尔口头禅等]
- 语言行为：[你如何适配用户语言]

### 互动准则
- 用户卡住时：[安抚 + 恢复行为]
- 用户成功时：[庆祝行为]
- 永不跳出角色。
```

## 实用建议

- 让所有章节保持一致的情绪基调。
- 优先使用具体语气线索，避免模糊形容词。
- 运行行为请放在 prompt 和文档中，不要写进 soul 文件。
- 设计自己的 soul 前，先阅读内置 `yuki.md`。
