---
title: "Fehlerbehebung"
description: "Behebe haeufige MineClawd-Setup- und Laufzeitprobleme schnell."
---

Wenn MineClawd fehlschlaegt, finde unten zuerst das naechste passende Symptom und wende diese Loesung an.

<AccordionGroup>
  <Accordion title="MineClawd sagt, nur OP-Nutzer koennen diesen Befehl ausfuehren">
    Gib deinem Spieler OP-Level 2. Jeder `/mineclawd`- und `/mclawd`-Befehl erfordert OP-Zugriff.
  </Accordion>

  <Accordion title="MineClawd sagt, API-Key oder Modell fehlt">
    Oeffne die Konfiguration und setze die Provider-Zugangsdaten.

    OpenAI-Pfad:
    - `provider openai`
    - `api-key <key>`
    - `model <model>`

    Vertex-Pfad:
    - `provider vertex-ai`
    - `vertex-api-key <key>`
    - `vertex-model <model>`
  </Accordion>

  <Accordion title="`/mineclawd config` oeffnet die GUI nicht">
    Dein Client hat MineClawd wahrscheinlich nicht installiert. Nutze die Konfiguration per Befehl:

    ```bash
    /mineclawd config <key> <value>
    ```
  </Accordion>

  <Accordion title="`/mineclawd history` sagt, Client-Mod ist erforderlich">
    Installiere MineClawd auf deinem Client. Die History-Book-UI verwendet einen Client-Paketkanal.
  </Accordion>

  <Accordion title="Eine andere Anfrage laeuft noch fuer diese Session">
    Warte, bis die aktuelle Aufgabe beendet ist, bevor du eine neue Anfrage startest, die Session wechselst oder die persona wechselst.
  </Accordion>

  <Accordion title="Retry-Token ist ungueltig oder abgelaufen">
    Retry-Tokens verfallen nach 30 Minuten.

    Starte eine neue Anfrage oder nutze die letzte Retry-Aktion aus einer frischen Fehlermeldung.
  </Accordion>

  <Accordion title="Session nicht gefunden fuer resume, remove oder repair">
    Fuehre zuerst `/mineclawd sessions list` aus.

    Nutze dann exakt die Session-ID oder das Token aus dieser Ausgabe.
  </Accordion>

  <Accordion title="Persona nicht gefunden">
    Pruefe verfuegbare Namen mit `/mineclawd persona`.

    Erstelle bei Bedarf eine neue persona-Datei unter `mineclawd/souls/`.
  </Accordion>

  <Accordion title="Dynamic placeholder registry ist deaktiviert">
    Setze den Modus und starte neu:

    ```bash
    /mineclawd config dynamic-registry-mode enabled
    ```

    Auf dedizierten Servern muessen Clients MineClawd ebenfalls installiert haben, wenn dies aktiviert ist.
  </Accordion>

  <Accordion title="`reload-game` meldet KubeJS-Fehler">
    Lies die gemeldeten Zeilen, behebe Syntax- oder Laufzeitprobleme in `kubejs/server_scripts/mineclawd/` und lade erneut.

    Wiederhole das, bis die Reload-Ausgabe keine KubeJS-Fehler mehr zeigt.
  </Accordion>

  <Accordion title="`/mineclawd choose` sagt, es gibt keine offene Frage">
    MineClawd wartet nicht auf eine Antwort.

    Fuehre `choose` nur aus, nachdem MineClawd eine Rueckfrage gestellt hat.
  </Accordion>

  <Accordion title="Es gibt nur eine Option bei LLM provider-Auswahl/Broadcast Requests To">
    Das ist ein bekannter YACL-Bug. Klicke auf den Wert und gib die ersten ein bis zwei Buchstaben der gewuenschten Option ein. Waehle dann die Option im Dropdown.

    Fuer LLM provider: tippe `o` fuer OpenAI oder `g` fuer Google Vertex AI.
    Fuer Broadcast Requests To: tippe `on` fuer only-me, `a` fuer all oder `op` fuer ops.
  </Accordion>

  <Accordion title="Der Overlay-Orb erscheint nicht">
    Der Assistive Touch Orb erfordert:

    - MineClawd auf dem Client.
    - OP-Berechtigung Level 2.
    - **Enable GUI** auf ON in der Konfiguration (**Mod Menu** -> **MineClawd** -> **Configure**).
    - Assistive Touch nicht deaktiviert (`/mineclawd assistivetouch` zum Reaktivieren).

    Pruefe jeden Punkt. Wenn der Orb weiterhin fehlt, aktiviere den Debug-Modus und pruefe die Logs.
  </Accordion>

  <Accordion title="Tippen im Overlay landet im Chat oder Inventar">
    Stelle sicher, dass du die neueste MineClawd-Version nutzt.

    Wenn das Problem bleibt, kann es ein Mod-Konflikt sein. Teste nur mit MineClawd und seinen Abhaengigkeiten.
  </Accordion>

  <Accordion title="Websuche sagt, das search-Tool ist deaktiviert">
    Das `search`-Tool benoetigt einen Tavily API-Key. Setze ihn mit:

    ```bash
    /mineclawd config tavily-api-key <your-tavily-key>
    ```

    Du bekommst einen Key auf [tavily.com](https://tavily.com/). Sobald konfiguriert, nutzt MineClawd die Websuche automatisch, wenn externe Informationen noetig sind.
  </Accordion>

  <Accordion title="Stop-Button oder /mineclawd stop bricht die Anfrage nicht ab">
    Stop sendet ein Abbruchsignal an die aktive Anfrage. Wenn der LLM-Provider bereits fertig gerechnet hat, die Antwort aber noch uebertragen wird, wirkt stop eventuell nicht sofort.

    Wenn die Session nach stop gesperrt bleibt, erstelle mit `/mineclawd sessions new` eine neue Session.
  </Accordion>

  <Accordion title="Asset teleport meldet, die Entity kann nicht gefunden werden">
    Die Entity wurde eventuell getoetet, despawned oder entladen. Asset-Eintraege speichern den letzten bekannten Ort zum Erstellungszeitpunkt und verfolgen keine Bewegung.

    Teleport funktioniert nur fuer Entity-Kategorie-Assets mit gespeicherter UUID und Dimension.
  </Accordion>
</AccordionGroup>

## Wenn Probleme bleiben

- Aktiviere Debug-Logs: `/mineclawd config debug-mode true`
- Reproduziere das Problem einmal
- Pruefe `logs/latest.log`
- Erstelle ein Issue: `https://github.com/Zhou-Shilin/MineClawd/issues`
