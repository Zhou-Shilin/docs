---
title: "retry"
description: "Reintenta la última solicitud fallida usando un token de reintento."
---

## Propósito

Repite una solicitud fallida sin volver a escribir el prompt.

## Sintaxis

```bash
/mineclawd retry <token>
```

## De dónde viene el token

Cuando una solicitud falla, MineClawd puede mostrar acciones clicables `[Retry]` y `[Adjust Prompt]`. Usa el token detrás de `[Retry]`.

## Comportamiento y límites

- Los tokens de reintento vencen después de 30 minutos.
- El reintento solo funciona cuando tu sesión activa aún coincide con la sesión original que falló.
- Si el reintento inicia correctamente, MineClawd limpia el token fallido almacenado.

## Errores comunes

- `retry token is invalid or expired`: la solicitud falló hace demasiado tiempo o el token es incorrecto.
- `active session changed`: reanuda la sesión original o vuelve a enviarla manualmente.
- `another request is still running for this session`: espera a que termine.
