## 目标

按本页步骤操作后，你将获得一次 MineClawd 的成功回复。

## 第 1 步：为 Minecraft 1.20.1 安装 Fabric

1. 从这里下载 Fabric Installer：[fabricmc.net/use/installer](https://fabricmc.net/use/installer/)
2. 运行安装程序。
3. 选择：
   - 安装类型：**Client**
   - Minecraft 版本：**1.20.1**
4. 点击 **Install**。
5. 打开 Minecraft Launcher，先运行一次 **Fabric Loader 1.20.1**，然后关闭游戏。

<Note>
在 Windows 和官方启动器环境下，这通常是一次点击即可完成的安装流程。
</Note>

## 第 2 步：下载必需模组（Fabric + 1.20.1 版本）

从 Modrinth 下载以下文件。每个页面都要选择满足这些条件的版本：

- Minecraft 版本：`1.20.1`
- 加载器：`Fabric`

| 模组 | 下载链接 |
| --- | --- |
| MineClawd | [modrinth.com/mod/mineclawd](https://modrinth.com/mod/mineclawd) |
| Fabric API | [modrinth.com/mod/fabric-api](https://modrinth.com/mod/fabric-api) |
| KubeJS | [modrinth.com/mod/kubejs](https://modrinth.com/mod/kubejs) |
| Architectury API | [modrinth.com/mod/architectury-api](https://modrinth.com/mod/architectury-api) |
| Rhino | [modrinth.com/mod/rhino](https://modrinth.com/mod/rhino) |
| YetAnotherConfigLib v3 (YACL) | [modrinth.com/mod/yacl](https://modrinth.com/mod/yacl) |
| Mod Menu (for GUI config) | [modrinth.com/mod/modmenu](https://modrinth.com/mod/modmenu) |

## 第 3 步：把模组文件放进 `mods` 文件夹

1. 打开你的 Minecraft 文件夹。
2. 打开 `mods` 文件夹。如果不存在就新建。
3. 把所有下载的 `.jar` 文件移动到 `mods` 中。
4. 不要解压 `.jar` 文件。

## 第 4 步：启动游戏并打开 MineClawd GUI

1. 使用 **Fabric Loader 1.20.1** 配置启动 Minecraft。
2. 创建或打开一个单人世界。
3. 为该世界开启 **Allow Cheats**。
4. 打开 **Mod Menu** -> **MineClawd** -> **Configure**。

## 第 5 步：连接 AI 服务

如果你还没有 AI 账号，可以先用 OpenRouter 并选择免费模型。

1. 创建账号：[openrouter.ai](https://openrouter.ai/)
2. 创建 API Key：[openrouter.ai/settings/keys](https://openrouter.ai/settings/keys)
3. 在 MineClawd GUI 中设置：
   - **Provider**: `OpenAI`
   - **Endpoint**: `https://openrouter.ai/api/v1`
   - **API Key**：粘贴你的 OpenRouter key
   - **Model**：从 [openrouter.ai/models](https://openrouter.ai/models) 选择一个 model id（免费模型通常带 `:free`）
   - **Summarize Model**：先使用相同的 model id
4. 保存配置。

<Info>
免费模型和限制会随时间变化。如果某个模型失败，请在同一模型页面选择另一个 model id。
</Info>

## 第 6 步：发送第一条消息

在你的世界里打开聊天并运行：

```bash
/mclawd 你好
```

如果配置正确，MineClawd 会在聊天栏回复。

## 如果没有成功

前往 [/resources/troubleshooting](/resources/troubleshooting)，按对应症状排查。
