---
title: "config"
description: "MineClawd の設定を開く・確認する・更新する。"
---

## 目的

このコマンドで GUI 設定を開くか、チャットから設定を管理します。

## 構文

```bash
/mineclawd config
/mineclawd config <key>
/mineclawd config <key> <value>
```

## よく使うキー

- `provider`: `openai` または `vertex-ai`
- `endpoint`, `api-key`, `model`, `summarize-model`
- `vertex-endpoint`, `vertex-api-key`, `vertex-model`, `vertex-summarize-model`
- `debug-mode`: `true` または `false`
- `limit-tool-calls`: `true` または `false`
- `tool-call-limit`: `1` から `20`
- `system-prompt`
- `broadcast-requests-to`: `self`, `all`, または `ops`
- `dynamic-registry-mode`: `auto`, `enabled`, または `disabled`

## 例

```bash
/mineclawd config provider openai
/mineclawd config api-key sk-...
/mineclawd config tool-call-limit 12
/mineclawd config broadcast-requests-to ops
/mineclawd config dynamic-registry-mode enabled
```

## 動作と制限

- `/mineclawd config` は、プレイヤークライアントに MineClawd が入っている場合のみ GUI を開きます。
- `broadcast-requests-to` はプレイヤーごとの設定で、ゲーム内で設定する必要があります。
- 秘密値は読み取り時にマスクされます。
- `dynamic-registry-mode` の変更後はゲームまたはサーバー再起動が必要です。

## よくある失敗

- `unknown config key`: スペルを確認するかタブ補完を使ってください。
- `invalid boolean`: `true` か `false` を使ってください。
- `tool-call-limit must be between 1 and 20`: 範囲内の値を指定してください。
