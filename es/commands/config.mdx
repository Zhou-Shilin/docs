---
title: "config"
description: "Abre, consulta o actualiza la configuración de MineClawd."
---

## Propósito

Usa este comando para abrir la configuración de la GUI o gestionar ajustes desde chat.

## Sintaxis

```bash
/mineclawd config
/mineclawd config <key>
/mineclawd config <key> <value>
```

## Claves comunes

- `provider`: `openai` o `vertex-ai`
- `endpoint`, `api-key`, `model`, `summarize-model`
- `vertex-endpoint`, `vertex-api-key`, `vertex-model`, `vertex-summarize-model`
- `debug-mode`: `true` o `false`
- `limit-tool-calls`: `true` o `false`
- `tool-call-limit`: `1` a `20`
- `system-prompt`
- `tavily-api-key`: Tavily API key para búsqueda web
- `broadcast-requests-to`: `self`, `all` u `ops`
- `dynamic-registry-mode`: `auto`, `enabled` o `disabled`

## Ejemplos

```bash
/mineclawd config provider openai
/mineclawd config api-key sk-...
/mineclawd config tool-call-limit 12
/mineclawd config broadcast-requests-to ops
/mineclawd config dynamic-registry-mode enabled
```

## Comportamiento y límites

- `/mineclawd config` abre la GUI solo cuando el cliente del jugador tiene MineClawd instalado.
- `broadcast-requests-to` es por jugador y debe definirse dentro del juego.
- Los valores secretos se enmascaran cuando los consultas.
- Cambiar `dynamic-registry-mode` requiere reiniciar juego o servidor.

## Fallos comunes

- `unknown config key`: revisa ortografía o usa autocompletado con tab.
- `invalid boolean`: usa `true` o `false`.
- `tool-call-limit must be between 1 and 20`: mantente en el rango.
