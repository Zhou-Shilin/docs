## Zweck

Diese Seite zeigt dir das komplette MineClawd-Konfigurationsmodell, inklusive Dateieinstellungen, Befehlschlüsseln und Neustartverhalten.

## Wo Einstellungen liegen

- Globale Konfigurationsdatei: `config/mineclawd.json5`
- Sichtbarkeit im Chat pro Spieler: `mineclawd/player-settings.json`
- Soul-Dateien und aktiver Soul-State: `mineclawd/souls/` und `mineclawd/souls/.active/`

## Vollständige `mineclawd.json5`-Vorlage

```json5
{
  provider: "OPENAI",
  endpoint: "https://api.openai.com/v1",
  apiKey: "",
  model: "gpt-4o-mini",
  summarizeModel: "gpt-4o-mini",

  vertexEndpoint: "https://aiplatform.googleapis.com/v1",
  vertexApiKey: "",
  vertexModel: "publishers/google/models/gemini-3-pro-preview",
  vertexSummarizeModel: "publishers/google/models/gemini-3-pro-preview",

  debugMode: false,
  limitToolCalls: false,
  toolCallLimit: 16,
  systemPrompt: "",
  dynamicRegistryMode: "AUTO",

  // Client-only (nicht in der Serverkonfiguration)
  enableGui: true
}
```

<Info>
In Dateiform sind Enum-Werte großgeschrieben (`OPENAI`, `VERTEX_AI`, `AUTO`, `ENABLED`, `DISABLED`). In Befehlsform sind sie kleingeschrieben (`openai`, `vertex-ai`, `auto`, `enabled`, `disabled`).
</Info>

## Schlüsselreferenz

| Dateischlüssel | Befehlschlüssel | Typ | Vorgabe | Hinweise |
| --- | --- | --- | --- | --- |
| `provider` | `provider` | enum | `OPENAI` | Provider-Auswahl: OpenAI-kompatibel oder Vertex AI. |
| `endpoint` | `endpoint` | string | `https://api.openai.com/v1` | OpenAI-kompatible Basis-URL, ohne abschließenden Slash. |
| `apiKey` | `api-key` | string | leer | OpenAI-API-Key. |
| `model` | `model` | string | `gpt-4o-mini` | OpenAI-Chatmodell. |
| `summarizeModel` | `summarize-model` | string | `gpt-4o-mini` | OpenAI-Modell für die erste Session-Titelerstellung. |
| `vertexEndpoint` | `vertex-endpoint` | string | `https://aiplatform.googleapis.com/v1` | Vertex-API-URL. |
| `vertexApiKey` | `vertex-api-key` | string | leer | Vertex-API-Key. |
| `vertexModel` | `vertex-model` | string | `publishers/google/models/gemini-3-pro-preview` | Vertex-Modellpfad oder Modellname. |
| `vertexSummarizeModel` | `vertex-summarize-model` | string | `publishers/google/models/gemini-3-pro-preview` | Vertex-Zusammenfassungsmodell für Session-Titel. |
| `debugMode` | `debug-mode` | boolean | `false` | Zeichnet detaillierte Request- und Tool-Aktivitäten auf. |
| `limitToolCalls` | `limit-tool-calls` | boolean | `false` | Aktiviert ein Tool-Loop-Limit pro Anfrage. |
| `toolCallLimit` | `tool-call-limit` | integer | `16` | Erlaubter Bereich: `1` bis `20`. |
| `systemPrompt` | `system-prompt` | string | leer | Leer bedeutet eingebaute Systemprompt. Verwende `default`, um den Wert im Befehlsmodus zurückzusetzen. |
| `dynamicRegistryMode` | `dynamic-registry-mode` | enum | `AUTO` | Steuert den Runtime-Modus für dynamische Platzhalter. Neustart erforderlich nach Änderung. |
| `enableGui` | -- | boolean | `true` | Nur Client-seitig. Zeigt das Overlay oder nicht. Über **Mod Menu** -> **MineClawd** -> **Configure** steuerbar. |

## Einstellungen pro Spieler

### broadcast-requests-to

- Befehlschlüssel: `broadcast-requests-to`
- Zulässige Werte: `self`, `all`, `ops`
- Gespeichert in: `mineclawd/player-settings.json`
- Zweck: Steuert, wer Prompt-Echo und Statusmeldungen sieht.

### assistivetouch

- Befehl: `/mineclawd assistivetouch`
- Gespeichert in: `mineclawd/player-settings.json`
- Standard: aktiviert (sichtbar)
- Zweck: Zeigt oder versteckt den schwebenden AssistiveTouch-Orb ohne komplette GUI-Abschaltung.

## Verhalten von dynamic registry mode

- `AUTO`: Aktiviert im Client-Runtime (Einzelspieler), deaktiviert auf dedizierten Servern.
- `ENABLED`: Erzwingt Aktivierung in allen Laufzeiten.
- `DISABLED`: Deaktiviert vollständig.

<Warning>
Wenn du `ENABLED` auf einem dedizierten Server erzwingst, müssen alle Clients MineClawd installiert haben.
</Warning>

## Empfohlene Baselines

### OpenAI-Erstsetup

```bash
/mineclawd config provider openai
/mineclawd config endpoint https://api.openai.com/v1
/mineclawd config api-key <your-key>
/mineclawd config model gpt-4o-mini
/mineclawd config summarize-model gpt-4o-mini
```

### Vertex-Erstsetup

```bash
/mineclawd config provider vertex-ai
/mineclawd config vertex-endpoint https://aiplatform.googleapis.com/v1
/mineclawd config vertex-api-key <your-key>
/mineclawd config vertex-model publishers/google/models/gemini-3-pro-preview
/mineclawd config vertex-summarize-model publishers/google/models/gemini-3-pro-preview
```

## GUI-Pfad

- Im Spiel: **Mod Menu** -> **MineClawd** -> **Configure**
- Als Befehl: `/mineclawd config ...`
- Im Overlay-Menü: Hamburger-Icon klicken -> **Config**

Wenn das GUI nicht öffnet, nutze die Befehls-Schlüssel.

<Info>
Die Einstellung `enableGui` steuert das Overlay. Wenn du sie auf OFF setzt, bleibt der YACL-Konfigurationsbildschirm erreichbar – du kannst **Mod Menu** oder `/mineclawd config` weiterhin verwenden.
</Info>
