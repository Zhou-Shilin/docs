---
title: "Solución de problemas"
description: "Resuelve rápido problemas comunes de configuración y ejecución de MineClawd."
---

Si MineClawd falla, busca abajo el síntoma más cercano y aplica primero esa solución.

<AccordionGroup>
  <Accordion title="MineClawd dice que solo los usuarios OP pueden ejecutar este comando">
    Da a tu jugador permisos OP de nivel 2. Todos los comandos `/mineclawd` y `/mclawd` requieren acceso OP.
  </Accordion>

  <Accordion title="MineClawd dice que falta la clave API o el modelo">
    Abre la configuración y define las credenciales del proveedor.

    Ruta OpenAI:
    - `provider openai`
    - `api-key <key>`
    - `model <model>`

    Ruta Vertex:
    - `provider vertex-ai`
    - `vertex-api-key <key>`
    - `vertex-model <model>`
  </Accordion>

  <Accordion title="`/mineclawd config` no abre la GUI">
    Es probable que tu cliente no tenga MineClawd instalado. Usa configuración por comando:

    ```bash
    /mineclawd config <key> <value>
    ```
  </Accordion>

  <Accordion title="`/mineclawd history` dice que se requiere el mod de cliente">
    Instala MineClawd en tu cliente. La interfaz de libro de historial usa un canal de paquetes de cliente.
  </Accordion>

  <Accordion title="Otra solicitud sigue ejecutándose para esta sesión">
    Espera a que termine la tarea actual antes de iniciar otra, cambiar de sesión o cambiar de personalidad.
  </Accordion>

  <Accordion title="El token de reintento no es válido o está vencido">
    Los tokens de reintento vencen a los 30 minutos.

    Lanza una nueva solicitud o usa la acción de reintento más reciente desde un mensaje de error nuevo.
  </Accordion>

  <Accordion title="No se encontró la sesión para reanudar, eliminar o reparar">
    Ejecuta primero `/mineclawd sessions list`.

    Luego usa el id de sesión o token exacto que aparece en esa salida.
  </Accordion>

  <Accordion title="No se encontró la personalidad">
    Comprueba los nombres disponibles con `/mineclawd persona`.

    Si falta un soul, crea un archivo nuevo en `mineclawd/souls/`.
  </Accordion>

  <Accordion title="El registro de marcadores dinámicos está deshabilitado">
    Configura el modo y reinicia:

    ```bash
    /mineclawd config dynamic-registry-mode enabled
    ```

    En servidores dedicados, los clientes también deben instalar MineClawd cuando esto esté habilitado.
  </Accordion>

  <Accordion title="`reload-game` informa errores de KubeJS">
    Lee las líneas reportadas y corrige errores de sintaxis o runtime en `kubejs/server_scripts/mineclawd/`, luego recarga.

    Repite hasta que la salida no muestre errores de KubeJS.
  </Accordion>

  <Accordion title="`/mineclawd choose` dice que no hay pregunta pendiente">
    MineClawd no está esperando una respuesta.

    Ejecuta `choose` solo después de que MineClawd haga una pregunta de aclaración.
  </Accordion>

  <Accordion title="Solo hay una opción en la selección de proveedor LLM/Broadcast Requests To">
    Es un bug extraño de YACL. Haz clic en el valor e introduce la(s) primera(s) letra(s) de la opción, luego selecciónala del desplegable.

    Para proveedor LLM: escribe `o` para OpenAI o `g` para Google Vertex AI.
    Para Broadcast Requests To: escribe `on` para only-me, `a` para all u `op` para ops.
  </Accordion>

  <Accordion title="El orbe de la superposición no aparece">
    El orbe AssistiveTouch requiere:

    - MineClawd instalado en el cliente.
    - Nivel de permiso OP 2.
    - **Enable GUI** activado en configuración (**Mod Menu** -> **MineClawd** -> **Configure**).
    - AssistiveTouch no está desactivado (`/mineclawd assistivetouch` para reactivarlo).

    Revisa cada punto. Si sigue sin aparecer, activa debug y consulta los registros.
  </Accordion>

  <Accordion title="Escribir en la superposición se filtra al chat o al inventario">
    Este problema se solucionó en v1.4.0. Asegúrate de usar MineClawd 1.4.0 o posterior.

    Si persiste, puede ser un conflicto con otros mods. Prueba con solo MineClawd y sus dependencias.
  </Accordion>

  <Accordion title="El botón Stop o `/mineclawd stop` no cancela la solicitud">
    Stop envía una señal de cancelación a la solicitud activa. Si el proveedor ya terminó pero aún se transmite la respuesta, puede tardar en cancelarse.

    Si la sesión sigue bloqueada, crea una nueva con `/mineclawd sessions new`.
  </Accordion>

  <Accordion title="La teletransportación de activos dice que no se encuentra la entidad">
    La entidad pudo haber muerto, desaparecido o descargado. Los registros guardan la última ubicación conocida al crear el activo y no rastrean movimientos.

    Teletransportar solo funciona con activos de categoría entidad que tienen UUID y dimensión válidos.
  </Accordion>
</AccordionGroup>

## Si los problemas continúan

- Activa logs de depuración: `/mineclawd config debug-mode true`
- Reproduce el problema una vez
- Revisa `logs/latest.log`
- Abre un issue: `https://github.com/Zhou-Shilin/MineClawd/issues`
