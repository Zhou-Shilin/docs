---
title: "retry"
description: "Die letzte fehlgeschlagene Anfrage mit einem Retry-Token erneut ausführen."
---

## Zweck

Eine fehlgeschlagene Anfrage erneut ausführen, ohne den Prompt neu zu tippen.

## Syntax

```bash
/mineclawd retry <token>
```

## Woher das Token kommt

Wenn eine Anfrage fehlschlägt, kann MineClawd klickbare Aktionen `[Retry]` und `[Adjust Prompt]` anzeigen. Nutze das Token hinter `[Retry]`.

## Verhalten und Grenzen

- Retry-Tokens laufen nach 30 Minuten ab.
- Retry funktioniert nur, wenn deine aktive Session noch mit der ursprünglichen fehlgeschlagenen Session übereinstimmt.
- Wenn Retry erfolgreich startet, löscht MineClawd das gespeicherte fehlgeschlagene Token.

## Häufige Fehler

- `retry token is invalid or expired`: Anfrage liegt zu lange zurück oder Token ist falsch.
- `active session changed`: ursprüngliche Session wieder aufnehmen oder manuell erneut senden.
- `another request is still running for this session`: auf Abschluss warten.
