---
title: "sessions resume"
description: "Cambia la sesión activa por id o token."
---

## Propósito

Activa de nuevo una sesión anterior para que las solicitudes nuevas continúen ese contexto.

## Sintaxis

```bash
/mineclawd sessions resume <session>
```

`<session>` puede ser:

- id de sesión, como `1a2b`
- token de sesión, como `1a2b-my-project-title`

## Ejemplos

```bash
/mineclawd sessions resume 1a2b
/mineclawd sessions resume 1a2b-my-project-title
```

## Comportamiento y límites

- Reanudar no edita el historial. Solo cambia el puntero de sesión activa.
- Usa primero `/mineclawd sessions list` si no estás seguro de qué referencia usar.

## Errores comunes

- `session not found`: usa el id o token exacto de la salida de la lista.
- `cannot switch sessions while a request is running`: espera a que termine.
