## Zweck

Diese Seite zeigt dir das komplette MineClawd-Konfigurationsmodell, inklusive Datei-Einstellungen, Befehls-keys und Neustartverhalten.

## Wo Einstellungen liegen

- Globale Konfigurationsdatei: `config/mineclawd.json5`
- Chat-Sichtbarkeit pro Spieler: `mineclawd/player-settings.json`
- persona-Dateien und aktiver persona-Status: `mineclawd/souls/` und `mineclawd/souls/.active/`

## Vollstaendige `mineclawd.json5`-Vorlage

```json5
{
  provider: "OPENAI",
  endpoint: "https://api.openai.com/v1",
  apiKey: "",
  model: "gpt-5.2",
  summarizeModel: "gpt-5.2",

  vertexEndpoint: "https://aiplatform.googleapis.com/v1",
  vertexApiKey: "",
  vertexModel: "gemini-3.1-pro-preview",
  vertexSummarizeModel: "gemini-3-flash-preview",

  tavilyApiKey: "",

  debugMode: false,
  limitToolCalls: false,
  toolCallLimit: 16,
  systemPrompt: "",
  dynamicRegistryMode: "AUTO",

  // Nur Client (nicht in der Server-Konfigurationsdatei)
  enableGui: true
}
```

<Info>
In Dateiform sind Enum-Werte gross geschrieben (`OPENAI`, `VERTEX_AI`, `AUTO`, `ENABLED`, `DISABLED`).

In Befehlsform sind Werte klein geschrieben (`openai`, `vertex-ai`, `auto`, `enabled`, `disabled`).
</Info>

## Key-Referenz

| Datei-key | Befehls-key | Typ | Default | Hinweise |
| --- | --- | --- | --- | --- |
| `provider` | `provider` | enum | `OPENAI` | Provider-Auswahl: OpenAI-kompatibel oder Vertex AI. |
| `endpoint` | `endpoint` | string | `https://api.openai.com/v1` | OpenAI-kompatible Basis-URL, kein abschliessender Slash noetig. |
| `apiKey` | `api-key` | string | leer | OpenAI API-Key. |
| `model` | `model` | string | `gpt-5.2` | OpenAI Chat-Modell. |
| `summarizeModel` | `summarize-model` | string | `gpt-5.2` | OpenAI-Modell fuer Session-Titelerstellung nach der ersten Runde. |
| `vertexEndpoint` | `vertex-endpoint` | string | `https://aiplatform.googleapis.com/v1` | Vertex API-Basis-URL. |
| `vertexApiKey` | `vertex-api-key` | string | leer | Vertex API-Key. |
| `vertexModel` | `vertex-model` | string | `gemini-3.1-pro-preview` | Vertex-Modellpfad oder Modellname. |
| `vertexSummarizeModel` | `vertex-summarize-model` | string | `gemini-3-flash-preview` | Vertex summarize-Modell fuer Session-Titel. |
| `tavilyApiKey` | `tavily-api-key` | string | leer | Tavily API-Key fuer Websuche. Wenn gesetzt, wird das `search`-Tool aktiviert. |
| `debugMode` | `debug-mode` | boolean | `false` | Loggt detaillierte Request- und Tool-Aktivitaeten. |
| `limitToolCalls` | `limit-tool-calls` | boolean | `false` | Aktiviert ein Tool-Loop-Limit pro Anfrage. |
| `toolCallLimit` | `tool-call-limit` | integer | `16` | Erlaubter Bereich: `1` bis `20`. |
| `systemPrompt` | `system-prompt` | string | leer | Leer bedeutet eingebauter Systemprompt. Nutze im Befehlsmodus den Wert `default` zum Zuruecksetzen. |
| `dynamicRegistryMode` | `dynamic-registry-mode` | enum | `AUTO` | Steuert den Runtime-Modus fuer dynamische Placeholder. Nach Aenderung ist ein Neustart noetig. |
| `enableGui` | -- | boolean | `true` | Nur Client-seitig. Zeigt oder versteckt die Overlay-GUI. Setze es ueber **Mod Menu** -> **MineClawd** -> **Configure**. |

## Einstellungen pro Spieler

### broadcast-requests-to

Nur per Befehl, pro Spieler.

- Befehls-key: `broadcast-requests-to`
- Erlaubte Werte: `self`, `all`, `ops`
- Gespeichert in: `mineclawd/player-settings.json`
- Zweck: steuert, wer Prompt-Echo und Task-Start- oder Task-Ende-Zeilen sieht.

### assistivetouch

Nur Toggle, pro Spieler.

- Befehl: `/mineclawd assistivetouch`
- Gespeichert in: `mineclawd/player-settings.json`
- Default: aktiviert (sichtbar)
- Zweck: zeigt oder versteckt den schwebenden Assistive Touch Orb, ohne die komplette GUI zu deaktivieren.

## Verhalten von dynamic registry mode

- `AUTO`: aktiviert in Client-Runtimes (Einzelspieler), deaktiviert auf dedizierten Servern.
- `ENABLED`: erzwingt Aktivierung in allen Runtimes.
- `DISABLED`: vollstaendig deaktiviert.

<Warning>
Wenn du `ENABLED` auf einem dedizierten Server erzwingst, muessen beitretende Clients MineClawd installiert haben.
</Warning>

## Empfohlene Baselines

### Baseline fuer erstes OpenAI-Setup

```bash
/mineclawd config provider openai
/mineclawd config endpoint https://api.openai.com/v1
/mineclawd config api-key <your-key>
/mineclawd config model gpt-5.2
/mineclawd config summarize-model gpt-5.2
```

### Baseline fuer erstes Vertex-Setup

```bash
/mineclawd config provider vertex-ai
/mineclawd config vertex-endpoint https://aiplatform.googleapis.com/v1
/mineclawd config vertex-api-key <your-key>
/mineclawd config vertex-model gemini-3.1-pro-preview
/mineclawd config vertex-summarize-model gemini-3-flash-preview
```

## GUI-Pfad

- Im Spiel: **Mod Menu** -> **MineClawd** -> **Configure**
- Befehls-Fallback: `/mineclawd config ...`
- Overlay-Menue: Hamburger-Icon klicken -> **Config**

Wenn das Oeffnen der Client-GUI fehlschlaegt, konfiguriere ueber Befehls-keys.

<Info>
Die Einstellung `enableGui` steuert das Overlay-Fenster. OFF beeinflusst den YACL-Konfigurationsbildschirm nicht. Du kannst die Konfiguration weiterhin ueber **Mod Menu** oder `/mineclawd config` oeffnen.
</Info>
