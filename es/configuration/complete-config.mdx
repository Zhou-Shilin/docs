## Propósito

Esta página te da el modelo completo de configuración de MineClawd, incluyendo ajustes de archivo, claves de comando y comportamiento de reinicio.

## Dónde viven los ajustes

- Archivo de configuración global: `config/mineclawd.json5`
- Visibilidad de chat por jugador: `mineclawd/player-settings.json`
- Archivos de persona (soul) y estado de soul activa: `mineclawd/souls/` y `mineclawd/souls/.active/`

## Plantilla completa de `mineclawd.json5`

```json5
{
  provider: "OPENAI",
  endpoint: "https://api.openai.com/v1",
  apiKey: "",
  model: "gpt-5.2",
  summarizeModel: "gpt-5.2",

  vertexEndpoint: "https://aiplatform.googleapis.com/v1",
  vertexApiKey: "",
  vertexModel: "gemini-3.1-pro-preview",
  vertexSummarizeModel: "gemini-3-flash-preview",

  tavilyApiKey: "",

  debugMode: false,
  limitToolCalls: false,
  toolCallLimit: 16,
  systemPrompt: "",
  dynamicRegistryMode: "AUTO",

  // Solo cliente (no va en el archivo de configuración del servidor)
  enableGui: true
}
```

<Info>
En formato de archivo, los valores enum van en mayúsculas (`OPENAI`, `VERTEX_AI`, `AUTO`, `ENABLED`, `DISABLED`).

En formato de comando, los valores van en minúsculas (`openai`, `vertex-ai`, `auto`, `enabled`, `disabled`).
</Info>

## Referencia de claves

| Clave de archivo | Clave de comando | Tipo | Predeterminado | Notas |
| --- | --- | --- | --- | --- |
| `provider` | `provider` | enum | `OPENAI` | Selección de proveedor: compatible con OpenAI o Vertex AI. |
| `endpoint` | `endpoint` | string | `https://api.openai.com/v1` | URL base compatible con OpenAI; no requiere barra final. |
| `apiKey` | `api-key` | string | vacío | OpenAI API key. |
| `model` | `model` | string | `gpt-5.2` | Modelo de chat OpenAI. |
| `summarizeModel` | `summarize-model` | string | `gpt-5.2` | Modelo OpenAI para generar el título de Session en el primer turno. |
| `vertexEndpoint` | `vertex-endpoint` | string | `https://aiplatform.googleapis.com/v1` | URL base de Vertex API. |
| `vertexApiKey` | `vertex-api-key` | string | vacío | Vertex API key. |
| `vertexModel` | `vertex-model` | string | `gemini-3.1-pro-preview` | Ruta de modelo Vertex o nombre de modelo. |
| `vertexSummarizeModel` | `vertex-summarize-model` | string | `gemini-3-flash-preview` | Modelo de resumen Vertex para títulos de Session. |
| `tavilyApiKey` | `tavily-api-key` | string | vacío | Tavily API key para búsqueda web. Cuando se define, habilita la herramienta `search`. |
| `debugMode` | `debug-mode` | boolean | `false` | Registra actividad detallada de solicitudes y herramientas. |
| `limitToolCalls` | `limit-tool-calls` | boolean | `false` | Habilita límite de loop de herramientas por solicitud. |
| `toolCallLimit` | `tool-call-limit` | integer | `16` | El rango permitido es `1` a `20`. |
| `systemPrompt` | `system-prompt` | string | vacío | Vacío usa el system prompt integrado. Usa valor `default` en modo comando para restablecer. |
| `dynamicRegistryMode` | `dynamic-registry-mode` | enum | `AUTO` | Controla el modo de runtime de placeholders dinámicos. Requiere reinicio tras cambiar. |
| `enableGui` | -- | boolean | `true` | Solo cliente. Muestra u oculta la GUI de superposición. Se define en **Mod Menu** -> **MineClawd** -> **Configure**. |

## Ajustes por jugador

### broadcast-requests-to

Solo comando, por jugador.

- Clave de comando: `broadcast-requests-to`
- Valores permitidos: `self`, `all`, `ops`
- Se guarda en: `mineclawd/player-settings.json`
- Propósito: controla quién ve el eco del prompt y líneas de inicio o fin de tarea.

### assistivetouch

Solo toggle, por jugador.

- Comando: `/mineclawd assistivetouch`
- Se guarda en: `mineclawd/player-settings.json`
- Predeterminado: habilitado (visible)
- Propósito: mostrar u ocultar el orbe Assistive Touch flotante sin desactivar todo el sistema GUI.

## Comportamiento de dynamic registry mode

- `AUTO`: habilitado en runtime de cliente (un jugador), deshabilitado en servidores dedicados.
- `ENABLED`: forzado como habilitado en todos los runtimes.
- `DISABLED`: deshabilitado por completo.

<Warning>
Si fuerzas `ENABLED` en un servidor dedicado, los clientes que se unan deben tener MineClawd instalado.
</Warning>

## Baselines recomendados

### Baseline para setup inicial con OpenAI

```bash
/mineclawd config provider openai
/mineclawd config endpoint https://api.openai.com/v1
/mineclawd config api-key <your-key>
/mineclawd config model gpt-5.2
/mineclawd config summarize-model gpt-5.2
```

### Baseline para setup inicial con Vertex

```bash
/mineclawd config provider vertex-ai
/mineclawd config vertex-endpoint https://aiplatform.googleapis.com/v1
/mineclawd config vertex-api-key <your-key>
/mineclawd config vertex-model gemini-3.1-pro-preview
/mineclawd config vertex-summarize-model gemini-3-flash-preview
```

## Ruta GUI

- En juego: **Mod Menu** -> **MineClawd** -> **Configure**
- Alternativa por comando: `/mineclawd config ...`
- Menú de superposición: haz clic en el icono de hamburguesa -> **Config**

Si abrir la GUI cliente falla, configura con claves de comando.

<Info>
El ajuste `enableGui` controla la ventana de superposición. Ponerlo en OFF no afecta la pantalla de configuración YACL: todavía puedes abrir configuración desde **Mod Menu** o `/mineclawd config`.
</Info>
