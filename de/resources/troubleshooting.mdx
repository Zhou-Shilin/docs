---
title: "Fehlerbehebung"
description: "Häufige MineClawd-Setup- und Laufzeitprobleme schnell beheben."
---

Wenn MineClawd fehlschlägt, suche unten das passendste Symptom und wende zuerst diesen Fix an.

<AccordionGroup>
  <Accordion title="MineClawd sagt, dass nur OP-Nutzer diesen Befehl ausführen können">
    Gib deinem Spieler OP-Berechtigung auf Stufe 2. Jeder `/mineclawd`- und `/mclawd`-Befehl benötigt OP-Zugriff.
  </Accordion>

  <Accordion title="MineClawd sagt, API-Key oder Modell fehlt">
    Öffne die Konfiguration und setze die Provider-Zugangsdaten.

    OpenAI-Weg:
    - `provider openai`
    - `api-key <key>`
    - `model <model>`

    Vertex-Weg:
    - `provider vertex-ai`
    - `vertex-api-key <key>`
    - `vertex-model <model>`
  </Accordion>

  <Accordion title="`/mineclawd config` öffnet die GUI nicht">
    Dein Client hat MineClawd wahrscheinlich nicht installiert. Nutze die Befehlskonfiguration:

    ```bash
    /mineclawd config <key> <value>
    ```
  </Accordion>

  <Accordion title="`/mineclawd history` meldet, dass ein Client-Mod erforderlich ist">
    Installiere MineClawd auf deinem Client. Die Verlauf-Buch-UI nutzt einen Client-Paketkanal.
  </Accordion>

  <Accordion title="Für diese Session läuft noch eine andere Anfrage">
    Warte auf den Abschluss der aktuellen Aufgabe, bevor du eine neue startest, Sessions wechselst oder die Persona wechselst.
  </Accordion>

  <Accordion title="Retry-Token ist ungültig oder abgelaufen">
    Retry-Tokens laufen nach 30 Minuten ab.

    Starte eine neue Anfrage oder nutze die neueste Retry-Aktion aus einer frischen Fehlermeldung.
  </Accordion>

  <Accordion title="Session zum Fortsetzen, Entfernen oder Reparieren nicht gefunden">
    Führe zuerst `/mineclawd sessions list` aus.

    Verwende dann die exakte Session-ID oder das exakte Token aus dieser Ausgabe.
  </Accordion>

  <Accordion title="Persona nicht gefunden">
    Prüfe verfügbare Namen mit `/mineclawd persona`.

    Erstelle bei Bedarf eine neue Soul-Datei unter `mineclawd/souls/`.
  </Accordion>

  <Accordion title="Dynamic-Placeholder-Registry ist deaktiviert">
    Setze den Modus und starte neu:

    ```bash
    /mineclawd config dynamic-registry-mode enabled
    ```

    Auf dedizierten Servern müssen Clients ebenfalls MineClawd installieren, wenn das aktiviert ist.
  </Accordion>

  <Accordion title="`reload-game` meldet KubeJS-Fehler">
    Lies die gemeldeten Zeilen, behebe Skript-Syntax- oder Laufzeitprobleme in `kubejs/server_scripts/mineclawd/` und lade dann erneut.

    Korrigiere weiter, bis die Reload-Ausgabe keine KubeJS-Fehler mehr meldet.
  </Accordion>

  <Accordion title="`/mineclawd choose` meldet keine ausstehende Frage">
    MineClawd wartet nicht auf eine Antwort.

    Führe `choose` nur aus, nachdem MineClawd eine Rückfrage gestellt hat.
  </Accordion>

  <Accordion title="In der LLM-Providerauswahl/Broadcast Requests To gibt es nur eine Option">
    Das ist ein merkwürdiger Bug von YACL. Klicke auf den Wert und gib den ersten (oder die ersten zwei) Buchstaben der gewünschten Option ein. Wähle sie dann aus dem erscheinenden Dropdown aus.

    Für den LLM-Provider: tippe `o` für OpenAI oder `g` für Google Vertex AI.
    Für Broadcast Requests To: tippe `on` für only-me, `a` für all oder `op` für ops.
  </Accordion>
</AccordionGroup>


## Wenn Probleme weiter bestehen

- Debug-Logs aktivieren: `/mineclawd config debug-mode true`
- Fehler einmal reproduzieren
- `logs/latest.log` prüfen
- Issue erstellen: `https://github.com/Zhou-Shilin/MineClawd/issues`
